# GTD — Getting Things Done

This is Shaul's reference for helping Limor manage herself using David Allen's GTD methodology.

---

## The GTD Philosophy

GTD is built on one idea: your brain is for having ideas, not holding them. Everything that has your attention needs to live in a trusted external system. When it does, your mind is clear and you can focus on actually doing things.

---

## The 5 Steps

### 1. Capture
Collect *everything* that has your attention — tasks, ideas, worries, things to buy, things to do, things to follow up on. Nothing lives in your head.

- Limor sends Shaul anything that's on her mind → Shaul captures it
- No filtering at this stage — capture first, clarify later

### 2. Clarify
For each captured item, ask:
- **Is it actionable?**
  - No → trash it, someday/maybe it, or file it as reference
  - Yes → what is the *very next physical action*?
- **Will it take less than 2 minutes?** → Do it now (don't capture it)
- **Can it be delegated?** → Delegate and track in Waiting For
- **Does it require multiple steps?** → It's a Project — define the next action

### 3. Organize
Everything goes into exactly one of these lists:

| List | What goes here |
|------|---------------|
| **Inbox** | Unclarified items (temporary) |
| **Next Actions** | Concrete next steps, sorted by context |
| **Projects** | Any outcome requiring 2+ actions |
| **Waiting For** | Delegated tasks / things you're waiting on |
| **Someday/Maybe** | Things you might do, not committed |
| **Calendar** | Time-specific or date-specific actions only |
| **Reference** | Non-actionable info worth keeping |

### 4. Reflect
- **Daily**: Scan your calendar and Next Actions list
- **Weekly Review** (the backbone of GTD):
  1. Clear your inbox to zero
  2. Review all projects — is there a next action for each?
  3. Review Waiting For — anything stuck?
  4. Review Someday/Maybe — anything to activate?
  5. Look at the week ahead

### 5. Engage
Do the work. When choosing what to do next, use:
- **Context** — where are you? what tools do you have?
- **Energy** — how sharp are you right now?
- **Time available** — how long do you have?
- **Priority** — what matters most?

## Contexts (Next Actions)

Contexts group actions by where/how they can be done:

| Context | Meaning |
|---------|---------|
| `@computer` | Needs a laptop/desktop |
| `@phone` | Calls or voice messages |
| `@errands` | Out and about |
| `@home` | Can only do at home |
| `@waiting` | Waiting on someone else |
| `@email` | Needs email |
| `@agenda:name` | To discuss with a specific person |

## Asana as GTD System

Shaul uses Asana to manage Limor's GTD lists:

| GTD List | Asana |
|----------|-------|
| Inbox | Asana Inbox / uncategorized tasks |
| Next Actions | Tasks with context tags, no section or "Next Actions" section |
| Projects | Asana Projects |
| Waiting For | Tasks tagged `waiting-for` or in "Waiting For" section |
| Someday/Maybe | Tasks in "Someday Maybe" project or tagged `someday` |
| Reference | Notes/descriptions on tasks or separate reference project |

## Shaul's GTD Behaviors

### When Limor says something is on her mind
→ Capture it immediately. Ask "is there a next action?" if it's unclear.

### When Limor shares a task or idea
→ Help clarify: actionable? project or single action? which context?

### Weekly Review (remind Limor on Sundays)
→ Walk through the 5-step review. Check Asana for:
- Projects without next actions
- Waiting For items older than a week
- Inbox items not yet processed

### Inbox Zero
→ When Limor asks to process her inbox, go through each item and clarify/organize it.

### The 2-minute rule
→ If something takes less than 2 minutes, flag it as "do now" instead of adding to a list.

### Project vs Task
→ If Limor mentions something that has multiple steps → create an Asana project. If it's one action → create a task.

## GTD Vocabulary (quick ref)

- **Next Action** — the very next physical step to move something forward
- **Project** — any outcome requiring 2+ actions (not necessarily big)
- **Someday/Maybe** — things you're not committing to but don't want to lose
- **Waiting For** — delegated or blocked items
- **Weekly Review** — the habit that keeps the whole system working
- **Capture** — getting it out of your head and into the system
- **Clarify** — deciding what something means and what (if anything) to do about it
- **Horizons of Focus** — GTD's altitude model: actions → projects → areas → goals → vision → life purpose
